eio_readahead

(PECL eio >= 0.0.1dev)

eio_readahead — Помещает данные из файла в кэш страницы

Описание

resource eio_readahead ( mixed $fd , int $offset , int $length [, int $pri = EIO_PRI_DEFAULT [, callable $callback = NULL [, mixed $data = NULL ]]] )

eio_readahead() заполняет кэш страницы данными из файла. Таким образом, дальнейшие обращения к этому файлу не будут влиять на работу дисковой подсистемы. Более детальную информацию можно получить на странице руководства READAHEAD(2).

Список параметров

fd

Поток, ресурс сокета или числовой файловый описатель.

offset

Позиция в файле, с которой будут читаться данные.

length

Количество считываемых байт.

pri

Приоритет запросов: EIO_PRI_DEFAULT, EIO_PRI_MIN, EIO_PRI_MAX, или NULL. Если передан NULL, то pri устанавливается в EIO_PRI_DEFAULT.

callback

Функция callback вызывается при завершении запроса. Она должна удовлетворять следующему прототипу:

void callback(mixed $data, int $result[, resource $req]);
data

является пользовательскими данными, переданными в запросе.

result

содержит результирующее значение, зависящее от запроса; обычно это значение, возвращаемое соответствующим системным вызовом.

req

является опциональным запрашиваемым ресурсом, который может использоваться с такими функциями как eio_get_last_error()

data

Данные, которые необходимо передать функции callback.

Возвращаемые значения

eio_readahead() возвращает ресурс запроса либо FALSE в случае ошибки.
